Angular like DOM abstraction

In the age of SPA, all UI frameworks or libraries manipulate the DOM in one way are the other to achieve desired output. Angular offers structural directives to manipulate the DOM, there are a few of them to meet various application needs, let's look at couple of them and mimic in plain Javascript.

  • ngFor
  • ngIf

NgFor Directive:

Imagine having a long list of items to display, it would be time taking and error prone to hand code them. Given a template expression angular generates elements for each data item provided and appends it to the DOM tree, well angular does more than that, keeps track of changes to the data item update, remove the elements when needed and much more. To keep it simple for the discussion, lets limit the scope to creation alone for now.

To being let's get to the questions that needs to be answered.
  1. We need to know what needs to be displayed? - list of items to display
  2. How will each item look like? - the template for an items for instance 'li'
  3. Where will it be displayed? - reference to an element with-in the document, for isntance body

const items = ["Pen", "Papper", "Ruler"];

NgIf Directive:

Given a boolean expression, when it evaluates to true or false NgIf adds or removes the elements from the DOM. This is helpful when you need to display or hide elements conditionally from the view. For instance when user searches from a large list of items, you may want to hide the elements that does not fit the serach query.
